# 2026-06-11 工作日志

## charmatch_demo.py — 字符级模板匹配算法验证 Demo (v2 重写)

基于 C++ charmatch 插件的底层数学逻辑（仿射变换、坐标映射反切、OK/NG 双列表判定），用 Python + PySide6 + OpenCV 编写了独立可运行的 Demo。

### v1 → v2 重写原因
用户反馈 v1 存在严重问题：
1. ROI 框无法自定义大小，拖拽角点时闪退（HandleItem 与父项的递归触发）
2. "设为基准模板"后调整参数无任何作用（信号连接不完善）
3. "保存字符模板图片"按钮始终不可点击（_chars_detected 状态未正确更新）
4. 整体工作流不流畅，Phase A/B 切换不够清晰

### v2 核心改进
1. **ROI 重写**：移除 HandleItem 子项方案，改用 RoiRectItem 自身的事件处理（hoverMoveEvent / mousePressEvent / mouseMoveEvent / mouseReleaseEvent），在矩形内部拖拽移动，在角落 10px 范围内拖拽缩放。绘制白色小方块标记四个角。
2. **信号连接修复**：所有参数 SpinBox 的 valueChanged 和 CheckBox 的 stateChanged 连接到 _on_param_changed，该函数在 template_set 状态下调用 _run_char_preview() 实时预览。
3. **状态同步**：_run_char_preview() 内正确设置 _chars_detected = len(char_rects) > 0，_update_ui_state() 据此启用/禁用按钮。
4. **工作流按钮**：① 加载模板图片 → ② 设为基准模板 → ③ 保存字符模板图片 → ④ 切换下一张(检测)，流程清晰，每步只有正确的按钮可点击。
5. ### v3 修复记录（2026-06-11 下午）

用户反馈 v2 仍有 5 个严重 Bug，逐一分析修复：

**Bug #1: 无法打开非 test_ 命名的图片**
- 根因：`cv2.imread()` 内部使用 C 的 `fopen()`，不支持 Unicode 路径
- 修复：新增 `_imread_unicode()` 静态方法，用 `np.fromfile(path) + cv2.imdecode()` 替代
- 应用位置：`_on_load_template()` 和 `_on_next_image()`

**Bug #2: ROI 框四个角出现黑色拖影**
- 根因：`paint()` 中绘制的角标方块超出 item 的 boundingRect，QGraphicsView 无法正确擦除旧位置
- 修复：移除 `paint()` 中的角标方块绘制，仅靠光标形状变化提示可拖拽缩放

**Bug #3: 检测框跳到画面左上角 + 参数调整无效果 + 按钮不可用（连环 Bug）**
- **根因 1（最致命）**：`_show_image()` 中 `self._scene.clear()` 销毁了所有 QGraphicsItem 的 C++ 对象，但 Python 引用 `self._align_roi` / `self._detect_roi` 仍指向已销毁的悬空指针。后续访问 `get_roi()` 返回错误坐标 → 检测框跳到错误位置
- **根因 2（get_roi 计算错误）**：`get_roi()` 用 `QReCtF(self.pos(), self.rect().size())` 只取了 pos，忽略了 rect 自身的本地偏移量。当 rect 为 `QRectF(50,60,...)` 且 pos=`(100,50)` 时，场景坐标为 `(150,110)` 但 get_roi 返回 `(100,50)` — 差了偏移量
- 修复：
  - 新增 `_save_roi_state()` / `_restore_roi()` 方法，在 `scene.clear()` 前保存 ROI 几何状态，clear 后重建全新的 RoiRectItem 实例
  - `get_roi()` 改为 `pos() + rect().topLeft()` 返回正确的场景坐标
- 连锁效应：ROI 位置正确 → `_run_char_preview()` 在正确的检测框内做字符切分 → `_chars_detected` 正确置 True → ③保存字符模板按钮可点 → 字符模板正常保存 → ④切换下一张按钮可点

**Bug #4: 没有保存参数的按钮**
- 新增功能：
  - 启动时自动加载 `charmatch_params.json`（如存在）
  - 每调整一个参数自动保存到 JSON 文件（静默）
  - 界面新增「💾 保存参数配置」和「📂 加载参数配置」两个按钮
  - 新增 `_apply_params_to_ui()` 方法（blockSignals 防止循环触发）

### 核心功能

1. **界面布局**：左图（QGraphicsView）+ 右控制面板（参数调节 + 操作按钮）
2. **双 ROI 交互**：定位框（蓝色，用于全局对齐）和检测框（黄色，用于字符检测），可拖拽/调整大小
3. **算法核心**：
   - 自适应二值化（ADAPTIVE_THRESH_MEAN_C，参数同 C++ 版）
   - 形态学闭运算
   - **轮廓空间距离聚类**（替代 C++ 的水平行列投影法）— 基于图连通分量 BFS，兼容中文字符偏旁部首合并和弧形排布
   - 模板匹配 TM_CCOEFF_NORMED
4. **Phase A**：基准模板建立 → 字符切分预览 → 保存字符模板
5. **Phase B**：模板匹配对齐 → ROI联动 → 字符匹配 → OK/NG 判定（绿色/红色框 + 界面状态文字）

### 关键技术决策

- 使用 `cluster_characters_by_distance()` 函数实现聚类，基于图连通分量算法
- 附加差异检测（对应 C++ drawContours 抹已匹配区域 + 再找残余轮廓）
- `align_by_template_match` 使用 cv2.matchTemplate（非 ORB，简化版）
- 所有算法函数在 C++ 源码中有对应关系，注释标注了映射行
